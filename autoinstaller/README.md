# 🚀 Nextcloud LEMP Autoinstaller (Ubuntu 24.04 + Self-Signed SSL)

[![Platform: Ubuntu](https://shields.io)](https://ubuntu.com)
[![Web Server: Nginx](https://shields.io)](https://nginx.org)
[![Database: MariaDB](https://shields.io)](https://mariadb.org)
[![Language: Python](https://shields.io)](https://python.org)
[![License: MIT](https://shields.io)](LICENSE)

Скрипт для полностью автоматической развертки персонального облачного хранилища [Nextcloud](https://nextcloud.com) на базе актуального веб-стека **LEMP** (**L**inux + **E**nginx + **M**ariaDB + **P**HP 8.3) с настроенным шифрованием трафика (HTTPS). 

Просто запустите скрипт на чистой системе — и через пару минут ваше приватное облако готово к работе.

---

## ✨ Возможности

* **Автоматизация «под ключ»**: Установка всех системных зависимостей, модулей PHP и утилит одной командой.
* **Безопасная СУБД**: Автоматическая инициализация MariaDB, создание изолированной базы данных и оптимизация безопасности (аналог `mysql_secure_installation`).
* **Оптимизация PHP 8.3**: Автоматический тюнинг `php.ini` под требования Nextcloud (увеличение `memory_limit`, `upload_max_filesize`, настройка `OPcache` и `JIT`).
* **Защищенный трафик (HTTPS)**: Автоматическая генерация самоподписанного SSL-сертификата со сроком действия 10 лет.
* **Умный конфиг Nginx**: Настройка веб-сервера с жестким редиректом `HTTP → HTTPS`, поддержкой `HTTP/2`, обработкой `index.php` и корректным роутингом веб-ссылок Nextcloud.
* **Тихая установка (Unattended)**: Развертывание самого Nextcloud через консольный интерфейс `occ` без необходимости проходить веб-мастер в браузере.
* **Фоновые задачи**: Автоматическое добавление системного Cron-задания для пользователя `www-data` (рекомендуемый режим работы Nextcloud).
* **Базовый файрвол**: Настройка и включение `UFW` с доступом только к необходимым портам (80, 443).

---

## 📋 Требования к системе

* **ОС**: Чистая установка **Ubuntu 24.04 LTS** (также поддерживается 22.04 LTS).
* **Права**: Доступ к учетной записи суперпользователя (**root**) или использование `sudo`.
* **Сеть**: Постоянное подключение к интернету для загрузки пакетов.
* **Настройка**: Рекомендуется заранее задать серверу статический IP-адрес.

## 🚀 Быстрый старт

### 1. Скачайте скрипт на сервер
Склонируйте репозиторий или скачайте напрямую файл развертывания:
```bash
wget https://raw.githubusercontent.com/ВАШ_АККАУНТ/nextcloud-autoinstall/main/install_nextcloud.py
```
*(Не забудьте заменить URL на актуальный адрес вашего репозитория)*

### 2. Сделайте файл исполняемым (опционально)
```bash
chmod +x install_nextcloud.py
```

### 3. Запустите установку
Скрипт вносит глобальные изменения в конфигурацию ОС, поэтому его запуск обязателен с правами суперпользователя:
```bash
sudo python3 install_nextcloud.py
```

### 4. Завершение работы
После успешного выполнения скрипт выведет в консоль финальные данные:
* 🌐 **URL для доступа**: `https://<IP_АДРЕС_ВАШЕГО_СЕРВЕРА>`
* 👤 **Логин администратора**: `admin` *(или ваш кастомный)*
* 🔑 **Пароль администратора**: сформированный пароль

> [!NOTE]
> При первом входе браузер выдаст предупреждение: *«Ваше подключение не защищено»*. Это стандартное поведение для самоподписанных SSL-сертификатов. Безопасно добавьте сайт в список исключений вашего браузера.

## ⚙️ Конфигурация переменных

Перед запуском скрипта вы можете открыть его в текстовом редакторе (`nano install_nextcloud.py`) и изменить дефолтные параметры в секции конфигурации:


| Переменная | Значение по умолчанию | Описание |
| :--- | :--- | :--- |
| `SERVER_IP` | `192.168.24.187` | Статический IP-адрес сервера (или доменное имя) |
| `DB_NAME` | `nextcloud` | Имя создаваемой базы данных в MariaDB |
| `DB_USER` | `nextclouduser` | Имя системного пользователя базы данных |
| `DB_PASS` | `StrongPassword123!` | Пароль от пользователя базы данных |
| `ADMIN_USER` | `admin` | Логин для входа в панель управления Nextcloud |
3. **Database Setup**: Удаление тестовых пользователей СУБД, создание пустой базы данных и делегирование прав пользователю `DB_USER`.
4. **Nextcloud Core**: Загрузка официального архива Nextcloud с сайта разработчиков, распаковка в директорию `/var/www/nextcloud` и выдача прав веб-серверу (`chown -R www-data:www-data`).
5. **PHP Tuning**: Автоматический поиск файлов конфигурации `php.ini` для `fpm` и `cli` с заменой директив памяти и таймаутов под требования Nextcloud.
6. **SSL Generation**: Создание приватного ключа и сертификата в изолированной директории `/etc/nginx/ssl/` с помощью `openssl`.
7. **Nginx Vhost**: Создание и активация виртуального хоста в `/etc/nginx/sites-available/` со специфичными правилами экранирования заголовков и путями для WebDAV.
8. **App Installation**: Вызов консольной утилиты `sudo -u www-data php occ maintenance:install` для записи структуры таблиц и создания учетной записи администратора.
9. **Post-Configuration**: Внесение IP-адреса в массив `trusted_domains` внутри `config.php`, добавление задания в системный `crontab` и активация `ufw`.

## 🔒 Рекомендации по безопасности

1. **Смена паролей**: Если вы развернули систему с дефолтными паролями, немедленно измените их в настройках профиля Nextcloud.
2. **Многофакторная аутентификация (2FA)**: После первого входа перейдите в *Настройки -> Безопасность* и активируйте TOTP (двухфакторную аутентификацию) для защиты учетной записи.
3. **Выход в глобальный интернет**: Если планируется открывать сервер наружу, замените самоподписанный SSL-сертификат на доверенный бесплатный сертификат от **Let's Encrypt** с помощью утилиты `certbot`, а также настройте проброс портов (Port Forwarding) на вашем роутере.
